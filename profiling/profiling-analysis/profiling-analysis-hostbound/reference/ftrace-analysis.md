# Host侧Trace分析

## 功能概述

通过解析系统trace文件，分析Host侧进程性能，识别存在问题的进线程及主要瓶颈。

## 支持的数据格式

- 系统trace文件（如Linux ftrace生成的trace文件）

## 分析内容

- **进程运行时间分析**：统计各进程的运行时间分布
- **上下文切换次数检测**：识别上下文切换过于频繁的进程
- **CPU迁移次数检测**：分析进程在CPU核心间的迁移情况
- **中断时间分析**：统计各进程的中断处理时间
- **问题进线程识别**：综合各项指标识别存在性能问题的进程

## 输出结果

- **存在问题的进线程列表**：标记出上下文切换过多等异常情况
- **各进程的性能指标**：运行时间、上下文切换次数、CPU迁移次数、中断时间
- **主要瓶颈分析**：指出系统性能瓶颈所在
- **Top 10 进程性能排序**：按运行时间排序的进程列表
- **中断时间最长的进程和中断类型**：分析中断对系统性能的影响

## 使用方式

```bash
python scripts/trace_analyzer.py --input <trace文件路径> [--output <报告输出路径>]
```

### 配置参数

| 参数 | 描述 | 默认值 |
|------|------|--------|
| `--input` | 系统trace文件路径 | 必填 |
| `--output` | 分析报告输出路径 | 在当前路径生成，以输入文件名命名 |

### 使用示例

```bash
# 基本Trace分析
python scripts/trace_analyzer.py --input "path\cpu_trace_all"

# 指定输出路径的Trace分析
python scripts/trace_analyzer.py \
  --input "path\cpu_trace_all" \
  --output "host_performance_analysis.xlsx"
```

## 输出结果解读

### 进程性能报告
生成包含以下内容的Excel报告：

- **进程基本信息**：PID、进程名、运行时间
- **性能指标**：上下文切换次数、CPU迁移次数、中断总时间
- **问题标记**：自动标记存在性能问题的进程
- **可视化图表**：便于直观分析进程性能分布和趋势

### 关键性能指标阈值

| 指标 | 正常范围 | 警告阈值 | 严重阈值 |
|------|----------|----------|----------|
| 上下文切换次数 | < 1,000次/秒 | 1,000-5,000次/秒 | > 5,000次/秒 |
| CPU迁移次数 | < 10次/秒 | 10-50次/秒 | > 50次/秒 |
| 中断时间占比 | < 1% | 1%-5% | > 5% |

## 常见问题分析

### 上下文切换过多
- **现象**：进程上下文切换次数远高于正常范围
- **影响**：CPU资源浪费、缓存失效、系统响应延迟
- **可能原因**：
  - 线程频繁创建和销毁
  - 严重的锁竞争或同步问题
  - 进程内部存在大量短周期任务
  - Python GIL（全局解释器锁）竞争

### CPU迁移频繁
- **现象**：进程在不同CPU核心间频繁切换
- **影响**：CPU缓存失效、性能下降
- **可能原因**：
  - 系统调度策略不当
  - 进程优先级设置不合理
  - 硬件中断分布不均匀

### 中断时间过长
- **现象**：进程中断处理时间占比较高
- **影响**：进程运行被频繁打断、实时性下降
- **可能原因**：
  - 硬件设备产生过多中断
  - 中断处理程序效率低下
  - 中断风暴（大量中断同时发生）
