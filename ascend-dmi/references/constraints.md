# Ascend DMI 使用约束和注意事项

> 来源: MindCluster 7.3.0 开发文档 - 昇腾社区

---

## 部署约束

- **独立部署**：Ascend DMI 工具必须独立部署在各服务器产品中使用
- **禁止共享目录**：不可部署在共享目录中供多台服务器共同访问使用
- **软件安装期间**：执行昇腾软件安装/升级/卸载时不可使用 Ascend DMI 工具

**违反后果**：功能执行失败或性能不符合预期等不可预测情况。

---

## 进程约束

- **单进程限制**：不支持在同一个设备里同时开启多个进程测试性能数据
- **空载要求**：建议在 NPU 空载、无业务的时候执行 P2P 压力测试

**多进程影响**：
- CCAE 或 npu-exporter 组件会占用通信链路带宽资源，导致结果误差
- CANN 性能分析工具采集数据时，Ascend DMI 不可使用
- 多进程测试可能导致测试结果不准确或失败

---

## 安全约束

执行 ascend-dmi 前，请确认以下指令安全可用：
- `fuser`
- `lscpu`
- `hccn_tool`
- `systemd-detect-virt`
- `dmidecode`
- `hostname`
- `mv`
- `cp`
- `gzip`

**建议**：提前安装 fuser 软件，以便 Ascend DMI 对 NPU 进程进行监测。

---

## 容器场景限制

以下功能在容器场景下执行可能失败：
- 码流测试
- 一键式组合诊断
- 一键式片上内存压测诊断
- AICORE 诊断/压测
- AICPU 压测
- 片上内存高危地址压测
- PRBS 码流诊断
- 眼图诊断/测试
- NPU 环境恢复

**故障表现**：plog 日志报错，驱动故障码为 46

**解决方案**：在宿主机上执行上述功能

---

## 混插卡限制

- Ascend DMI 不支持混插卡等非标形态产品
- 可能导致功能执行失败或性能不符合预期

---

## 日志级别要求

### 测试前确认

为避免频繁输出日志影响测试结果，测试前确认 Host 和 Device 日志级别为 **ERROR**。

### 确认方法

**Host 侧**：
```bash
# 查询日志级别
echo $GLOBAL_LOG_LEVEL
# 非法值或空表示缺省级别 ERROR（对应数值 3）

# 关闭 plog EVENT 日志
export ASCEND_GLOBAL_EVENT_ENABLE=0
```

**Device 侧**：
参考《msnpureport 工具使用》查看全局日志级别、模块日志级别和是否开启 Event 日志。

### 设置方法

如日志级别不为 ERROR，请参考官方文档设置 Host 和 Device 日志级别。

---

## 温度控制要求

### 算力测试

- 昇腾 AI 处理器预设性能阈值
- 建议在设备温度稳定并 **低于 90°C** 时进行算力测试
- 避免因温度过高触发主动降频，影响测试结果

### 功耗测试

- 测试时温度应正常范围内
- 如温度异常高，检查散热条件

---

## P2P 测试特殊约束

### 设备支持

P2P 带宽测试仅支持：
- Atlas 训练系列产品
- Atlas A2 训练/推理系列产品
- Atlas A2 推理系列产品
- Atlas 300I Duo 推理卡
- Atlas A3 训练/推理系列产品

### Atlas 300I Duo 特殊说明

- P2P 测试时，`-ds` 和 `-dd` 参数仅支持主芯片的 Device ID 或同一张卡的主从芯片对应的 Device ID

### Atlas 200T A2 Box16 / Atlas 200I A2 Box16 特殊说明

- 执行 P2P 测试时，仅支持输出两个对等位置的结果
- 例如：0 卡对应 8 卡，可输出 0→8 的 P2P 结果，以此类推

---

## D2D 带宽测试约束

### 不支持 -s 参数的产品

- Atlas A2 训练系列产品
- Atlas A2 推理系列产品
- Atlas A3 训练系列产品
- Atlas A3 推理系列产品

**原因**：D2D 传输数据大小由 AI Core 决定。`-et` 参数也不生效，默认为 1。

### 固定数据大小产品

- Atlas 200I/500 A2 推理产品：仅支持 D2D 模式，固定 0.97GB（由传输数据的 Tensor 决定），不支持 `-s` 和 `-et` 参数

---

## P2P 带宽测试参数交互约束

- 不指定 `-ds` 和 `-dd` 时：`-s`、`-et`、`-fmt` 参数**全部被忽略**，工具使用默认定长模式（A2 训练系列 Card 0/Card 8 默认 512MB，其他卡 256MB）
- `-ds` 和 `-dd` 必须成对使用，不能单独指定
- `-sp` 参数用于指定 P2P 模式下的特殊参数

---

## 算力测试数据类型支持

不同产品支持的算力测试类型不同：

| 产品 | fp16 | fp32 | hf32 | bf16 | int8 |
|-----|------|------|------|------|------|
| Atlas A2 训练系列 | ✓ | ✓ | ✓ | ✓ | ✓ |
| Atlas A3 训练系列 | ✓ | ✓ | ✓ | ✓ | ✓ |
| Atlas A2 推理系列 | ✓ | ✓ | ✓ | ✓ | ✓ |
| Atlas 训练系列（非A2） | ✓ | ✗ | ✗ | ✗ | ✓ |
| Atlas 200/300/500 推理 | ✓ | ✗ | ✗ | ✗ | ✓ |

不支持的类型指定后会报错。默认类型为 fp16。

---

## 诊断测试项组合约束

- `-i aicpu`（AICPU 压测）**不能与其他诊断项组合**，必须单独执行
- `-i random`（随机测试）**不能与其他诊断项组合**
- 压力测试中的 EDP/TDP：Atlas A3 系列**不能指定单个设备**执行

---

## 片上内存诊断命名差异

不同产品使用不同的参数名称：

| 参数名 | 适用产品 |
|-------|---------|
| `hbm` | Atlas A2/A3 系列产品 |
| `chipMemory` | Atlas 300I Pro / 300V / 300V Pro / 300I Duo |

两个名称**不能同时指定**。

---

## 设备 ID 说明

- 本文档输入/输出的 Device ID 都是**芯片逻辑 ID**
- 获取逻辑 ID：执行 `npu-smi info -m`，查看 Chip Logic ID 列
- NPU ID 为对应的芯片物理 ID

---

## 结果保存路径

### 默认路径

- **root 用户**：`/var/log/ascend_check`
- **非 root 用户**：`$HOME/var/log/ascend_check`

### 自定义路径

- 使用 `-r, --result` 参数指定
- 路径需符合安全要求
- 不支持包含通配符 "*"

### 权限设置

基于安全考虑，可将结果保存目录 `ascend_check` 权限设置为 700。

## 数据大小范围

### 带宽测试 -s 参数

| 产品 | 范围 | 说明 |
|-----|------|-----|
| Atlas A3 系列 | 1B ~ 4G | d2h/h2d/p2p 模式 |
| 其他产品 | 1B ~ 512M | - |

### 执行次数 --et 参数

- 取值范围：**[1, 1000]**
- 步长模式默认值：5
- 定长模式默认值：40

### 功耗测试时长 --dur 参数

- 取值范围：**[60, 604800]** 秒
- 默认值：600 秒

### 算力测试执行次数 --et 参数

- 取值范围：**[10, 80]**（单位：十万次）
- 默认值：60
