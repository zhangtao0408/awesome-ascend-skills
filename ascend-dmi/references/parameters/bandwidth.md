# 带宽测试参数说明

> 来源: MindCluster 7.3.0 开发文档 - 昇腾社区

---

## 主要参数

| 参数 | 说明 | 是否必填 | 默认值 | 约束 |
|-----|------|---------|-------|------|
| `-bw, --bw, --bandwidth` | 启用带宽测试 | 是 | - | - |
| `-t, --type` | 测试数据流向 | 否 | 返回 h2d/d2h/d2d 三个方向 | 见下方说明 |
| `-d, --device` | 指定设备 ID | 否 | Device 0（A3 系列返回全量） | 芯片逻辑 ID |
| `-s, --size` | 传输数据大小 | 否 | 步长模式 | 见下方说明 |
| `--et, --execute-times` | 迭代次数 | 否 | 步长 5 / 定长 40 | [1, 1000] |
| `-ds, --device-src` | P2P 源设备 | 否（p2p 时） | 全量测试 | 与 -dd 成对使用 |
| `-dd, --device-dst` | P2P 目标设备 | 否（p2p 时） | 全量测试 | 与 -ds 成对使用 |
| `-q, --quiet` | 跳过确认提示 | 否 | 提示确认 | - |
| `-fmt, --format` | 输出格式 | 否 | normal | normal/json |

---

## 测试类型 (-t) 说明

| 类型 | 说明 | 支持产品 | 特殊说明 |
|-----|------|---------|---------|
| `h2d` | Host 内存 → Device 内存 | 全系列 | 测试 PCIe 带宽 |
| `d2h` | Device 内存 → Host 内存 | 全系列 | 测试 PCIe 带宽 |
| `d2d` | Device 内存内部拷贝 | 全系列 | 测试内存带宽，A2/A3 系列不支持 -s |
| `p2p` | 设备间点对点传输 | 训练系列、A2/A3、300I Duo | 需指定源和目标或不指定测全量 |

**A3 系列特殊说明**：
- h2d/d2h 模式：测试 HCCS 平面总带宽

---

## 数据大小 (-s) 说明

### 支持范围

| 产品 | h2d/d2h/p2p | d2d |
|-----|-------------|-----|
| Atlas A3 系列 | 1B ~ 4G | 不支持 -s |
| 其他产品 | 1B ~ 512M | - |

### 模式说明

| 模式 | 触发条件 | 说明 |
|-----|---------|------|
| 步长模式 | 不指定 -s | 数据大小 2B ~ 32M，自动递增 |
| 定长模式 | 指定 -s | 固定数据大小，执行多次 |

### p2p 特殊默认值

| 产品 | 默认数据大小 |
|-----|-------------|
| A2/A3 系列，0/8 卡到其他卡 | 512M |
| A2/A3 系列，其他卡 | 256M |
| 其他产品 | 128M |

---

## 设备 ID 说明

- 本文档使用**芯片逻辑 ID**
- 获取方式：`npu-smi info -m`，查看 Chip Logic ID 列
- 多设备：逗号分隔（如 `0,1,2`）

---

## P2P 测试特殊参数

### 指定源和目标

```bash
ascend-dmi --bw -t p2p --ds <源设备> --dd <目标设备> [其他参数]
```

- `-ds` 和 `-dd` 必须成对使用
- 两参数值不能相同

### 不指定源和目标

```bash
ascend-dmi --bw -t p2p
```

- 测试全量昇腾 NPU 芯片
- 输出带宽矩阵

### 不支持 P2P 的产品

- Atlas 200/300/500 推理产品
- Atlas 300I Pro/V/V Pro 推理卡
- Atlas 200I SoC A1 核心板
- Atlas 200I/500 A2 推理产品

## 常用参数组合

### 基础测试（步长模式）

```bash
# 默认 h2d/d2h/d2d 全测
ascend-dmi --bw -q

# 指定设备 0 的 d2d 测试
ascend-dmi --bw -t d2d -d 0 -q
```

### 定长模式测试

```bash
# h2d 定长 128M，迭代 100 次
ascend-dmi --bw -t h2d -d 0 -s 134217728 --et 100 -q

# d2h 定长 128M，迭代 100 次
ascend-dmi --bw -t d2h -d 0 -s 134217728 --et 100 -q
```

### P2P 测试

```bash
# 全量 P2P 矩阵
ascend-dmi --bw -t p2p -q

# 指定 0→1 P2P 测试
ascend-dmi --bw -t p2p --ds 0 --dd 1 -s 128M --et 100 -q
```

## 输出字段

| 字段 | 说明 |
|-----|------|
| Size(Bytes)/Size(GB) | 传输数据大小 |
| Execute Times | 迭代次数 |
| Bandwidth(GB/s) | 带宽值 |
| Elapsed Time(us) | 耗时 |
