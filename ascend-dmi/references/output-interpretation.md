# Ascend DMI 输出结果解读指南

> 来源: MindCluster 7.3.0 开发文档 - 昇腾社区

---

## 诊断结果状态码

### 通用状态

| 状态 | 含义 | 处理建议 |
|-----|------|---------|
| **PASS** | 测试/诊断通过 | 设备正常，无需处理 |
| **FAIL** | 测试/诊断失败 | 需检查错误信息并处理 |
| **SKIP** | 当前产品/场景不支持 | 无需处理，属正常现象 |
| **HEALTH** | 健康状态（驱动/芯片） | 设备正常 |

### 警告级别（WARN 细分）

| 状态 | 含义 | 处理建议 |
|-----|------|---------|
| **GENERAL_WARN** | 一般警告 | 可继续使用，关注后续变化 |
| **IMPORTANT_WARN** | 重要警告 | 建议检查并处理 |
| **EMERGENCY_WARN** | 紧急警告 | **建议更换备件** |

## 带宽测试结果解读

### 输出字段说明

| 字段 | 说明 | 单位 |
|-----|------|-----|
| Size/Size(Bytes)/Size(GB) | 传输数据大小 | Bytes 或 GB |
| Execute Times | 迭代次数（内存拷贝次数） | 次 |
| Bandwidth(GB/s) | 带宽值 | GB/s |
| Elapsed Time(us) | 总执行时长 | 微秒 |

### 输出模式差异

- **步长模式**（默认，不加 `-s`）：输出多行，数据大小从 2B 递增到 32MB，每个大小一行结果
- **定长模式**（加 `-s SIZE`）：输出单行，固定指定大小，默认迭代 40 次
- **P2P 矩阵模式**（不指定 `-ds`/`-dd`）：输出设备间带宽矩阵，对角线显示 `**`（自身到自身），分别展示单向和双向带宽

### 默认设备行为（未指定 `-d`）

| 产品 | h2d/d2h/d2d 默认 | p2p 默认 |
|-----|-----------------|---------|
| Atlas A3 系列 | 返回**所有设备**的结果 | 返回所有设备间矩阵 |
| 其他产品 | 仅返回 **Device 0** | 仅返回 Device 0 |

### D2D 特殊约束

- **A2/A3 系列**：D2D 传输大小由 AI Core 决定，不支持 `-s` 参数，`-et` 默认为 1
- **Atlas 200I/500 A2**：仅支持 D2D 模式，固定 0.97GB 传输，不支持 `-s` 和 `-et`

### P2P 参数交互

- 不指定 `-ds`/`-dd` 时：`-s`、`-et`、`-fmt` 参数**全部被忽略**，使用默认定长模式
- `-ds` 和 `-dd` 必须**成对使用**，不能单独指定一个

### 正常参考值（典型值，因硬件而异）

| 测试类型 | 典型带宽范围 | 说明 |
|---------|-------------|------|
| H2D (Host to Device) | 20-28 GB/s | PCIe 带宽 |
| D2H (Device to Host) | 25-30 GB/s | PCIe 带宽 |
| D2D (Device to Device) | 760-800 GB/s | 内存带宽（A2/A3 由 AI Core 决定，值可能不同） |
| P2P (Peer to Peer) | 25-55 GB/s | 卡间互连带宽（并非所有产品支持） |

### 异常判断

- 带宽显著低于参考值（< 50%）：可能存在硬件问题或配置异常
- 带宽波动大：检查是否有其他进程占用 NPU，CCAE/npu-exporter 会占用通信链路带宽
- D2D 结果与预期不符：确认产品是否支持自定义数据大小
- P2P 测试失败或无结果：确认产品是否支持 P2P（部分推理产品不支持）
- 测试失败：检查设备状态和连接

## 算力测试结果解读

### 输出字段说明

| 字段 | 说明 | 单位 |
|-----|------|-----|
| Device | 设备逻辑 ID（非物理 ID），使用 `--all` 时显示为 `all` | - |
| Execute Times | 执行次数 | 次 |
| Duration(ms) | 执行耗时 | 毫秒 |
| TFLOPS@FP16 | FP16 算力值 | TFLOPS |
| TFLOPS@FP32 | FP32 算力值 | TFLOPS |
| TFLOPS@HF32 | HF32 算力值 | TFLOPS |
| TFLOPS@BF16 | BF16 算力值 | TFLOPS |
| TOPS@INT8 | INT8 算力值 | TOPS |
| Power(W) | 满算力实时功耗（采样周期性，短时测试可能波动，不代表精确功耗） | 瓦特 |

### `--all` 参数对输出的影响

使用 `--all` 时，输出含义与单卡测试**完全不同**：

| 字段 | 单卡测试（-d N） | 整机测试（--all） |
|-----|----------------|-----------------|
| TFLOPS/TOPS | 该卡的算力值 | **所有卡算力的总和** |
| Execute Times | 该卡的执行次数 | 所有卡执行次数的**平均值** |
| Duration(ms) | 该卡的执行耗时 | 所有卡耗时的**平均值** |
| Power(W) | 该卡的实时功耗 | 所有卡功耗的**平均值** |

判断整机算力是否达标时：整机 TFLOPS 应约等于 单卡参考值 × 卡数。

### Execute Times 含义差异

- **A2/A3 训练系列**：Execute Times = 单核矩阵乘执行次数 × AI Core 数 + 单核向量乘执行次数 × Vector Core 数
- **其他产品**：Execute Times = 单核矩阵乘执行次数 × AI Core 数

### 正常参考值（Atlas A2 训练系列示例，单卡）

| 数据类型 | 典型算力值 | 说明 |
|---------|-----------|------|
| FP16 | 300+ TFLOPS | 正常范围 |
| FP32 | 150+ TFLOPS | 仅 A2/A3 支持 |
| HF32 | 140+ TFLOPS | 仅 A2/A3 支持 |
| BF16 | 700+ TFLOPS | 仅 A2/A3 支持（A3 系列更高） |
| INT8 | 280+ TOPS | 正常范围 |

注意：以上为**单卡**参考值。使用 `--all` 时需乘以卡数来对比。不同产品系列规格差异大，以产品手册为准。

### 异常判断

- 算力显著低于参考值：可能触发降频（温度 > 90°C）或硬件异常
- Power 显示波动：正常现象，功耗按周期采样，测试时间短时前后两次采集间存在时间间隔导致数据波动。精确功耗请使用专用功耗测试（`-p`）
- 测试失败：检查设备状态和驱动版本

## 功耗测试结果解读

### 输出字段说明

| 字段 | 说明 | 单位 | 正常范围 |
|-----|------|-----|---------|
| Power | 实时功耗 | W | 根据产品规格 |
| Temperature | 芯片温度 | °C | < 90°C |
| AI Core Usage | AI Core 利用率 | % | 测试时接近 100% |
| Voltage | 芯片电压 | V | 0.7-0.8V |
| Frequency | 芯片频率 | MHz | 根据产品规格 |
| Health | 健康状态 | - | OK |

### 典型产品功耗参考

| 产品类型 | 典型功耗范围 | 说明 |
|---------|-------------|------|
| Atlas A2 训练系列 | 350W 左右 | 单芯片 |
| Atlas 300I Duo | 145W 左右 | 整卡 |

### 异常判断

- 温度过高（> 90°C）：可能触发降频，检查散热
- 功耗异常低：检查是否正确运行测试
- Health 非 OK：检查芯片状态

## 诊断测试结果解读

### healthCheck（健康检查）场景

| 检查项 | PASS 含义 | FAIL 可能原因 |
|-------|----------|--------------|
| **driver** | 驱动固件正常 | 驱动或固件安装异常 |
| **cann** | CANN 软件检测正常 | CANN 包安装异常、驱动安装异常 |
| **device** | Device 检测结果健康 | 芯片故障 |
| **network** | 网络检测结果健康 | 网络配置异常 |
| **hbm/chipMemory** | 片上内存检测通过 | 内存故障 |
| **signalQuality** | 信号质量正常 | PCIe/HCCS/RoCE 信号异常 |

### healthCheck 警告状态详情

#### 片上内存诊断警告

| 警告级别 | 触发条件 | 建议 |
|---------|---------|------|
| GENERAL_WARN | 历史多比特隔离页，告警码 0x80E18401；实时隔离页 >=16 且 <64 | 可继续使用 |
| EMERGENCY_WARN | 告警码 0x80E18402；隔离行过多；多比特实时隔离页 >=64 | **建议更换备件** |

#### SignalQuality 警告

| 警告级别 | 含义 | 建议 |
|---------|------|------|
| IMPORTANT_WARN | PCIe、HCCS、RoCE 信号质量异常 | 联系华为工程师 |

### performanceCheck（性能规格）场景

| 检查项 | PASS 含义 | FAIL 可能原因 |
|-------|----------|--------------|
| **aiflops** | 算力测试结果正常（大于参考值） | 算力低于参考值、触发过温降频 |
| **bandwidth** | 带宽测试结果正常 | 带宽低于参考值 |
| **nic** | NPU 网口连通性正常、带宽达标 | 网口 DOWN、未连通、带宽不达标 |

### stressTest（压力测试）场景

| 检查项 | PASS 含义 | FAIL/警告处理 |
|-------|----------|--------------|
| **aicore** | 诊断结果无异常 | EMERGENCY_WARN：建议更换硬件 |
| **hbmStress** | 片上内存压测通过 | FAIL：有新增多比特隔离页 |
| **bandwidthStress** | P2P 压测通过 | EMERGENCY_WARN：建议更换硬件 |
| **edp/tdp** | 功耗压测通过 | IMPORTANT_WARN：检查芯片告警 |
| **aicpu** | AICPU 压测通过 | EMERGENCY_WARN：建议更换硬件 |

## 设备状态查询结果解读

### 基本信息 (-b)

| 字段 | 正常值 | 异常判断 |
|-----|-------|---------|
| Health | OK | 非 OK 需检查 |
| Temperature | < 90°C | >= 90°C 警告 |
| AI Core Usage | 根据业务 | - |
| Used Memory | 根据业务 | 接近 100% 需注意 |

### 详细信息 (--dt)

| 字段 | 说明 | 关注要点 |
|-----|------|---------|
| AI Core Usage | AI Core 利用率 | - |
| Memory Bandwidth Usage | 内存带宽使用率 | 高负载时关注 |
| Power (Real-time/Rated) | 实时/额定功耗 | 对比是否接近额定值 |
| ECC Information | ECC 错误信息 | 关注错误计数 |
| PCIe LnkSta Speed/Width | 链路当前速率/带宽 | 对比 LnkCap 检查是否降速 |

### ECC 错误解读

| 错误类型 | 说明 | 处理建议 |
|---------|------|---------|
| Single-Bit Error | 单比特错误 | 可纠正，关注数量增长 |
| Double-Bit Error | 双比特错误 | 不可纠正，触发隔离 |

## 常见错误码参考

### 驱动相关

| 错误码 | 含义 | 处理 |
|-------|------|------|
| 0x80E18401 | 多比特历史隔离页告警 | 一般警告，可继续使用 |
| 0x80E18402 | 多比特隔离页过多 | 紧急警告，建议更换 |
| 0x80DF8402 | ECC 多比特实时隔离页过多 | 紧急警告，建议更换 |

### 故障排查一般步骤

1. 检查驱动状态：`npu-smi info`
2. 检查进程占用：`fuser -v /dev/davinci*`
3. 查看日志：`/var/log/ascend_seclog/`
4. 确认日志级别：Host 和 Device 设为 ERROR
5. 重新插拔或复位设备（必要时）
