# Conda Notes

当前版本不负责创建、修复或切换服务器上的 conda 环境。

## 选择规则

1. 如果使用者明确要使用服务器 conda 环境，必须提供 conda 环境名；不提供则不能使用服务器 conda 环境做测试。
2. 如果使用者提供了 conda 环境名，优先使用该环境名进行检查和测试。
3. 如果使用者没有提供 conda 环境名，则只能把“服务器上是否存在可用 conda 环境”作为候选信息返回给使用者确认，不能擅自选定。

## 与 CANN 的关系

- 如果最终使用 conda 环境，则默认使用服务器现有环境。
- 如果使用者同时对 CANN 版本有要求，需要先按 [cann](cann.md) 的规则检查服务器常见路径当前引用的是哪个 CANN 版本，再明确告知使用者。
- 如果使用者自行提供了 CANN 目录，则优先使用该目录，而不是常见路径自动检索结果。

## 不满足条件时

- 找不到指定 conda 环境：停止并要求使用者提供新的 conda 环境名或改用容器。
- conda 环境可用但 CANN 不满足要求：停止并要求使用者提供新的环境。
- 使用者既不提供 conda 环境名，也不接受候选 conda 环境：不要继续走 conda 方案。
