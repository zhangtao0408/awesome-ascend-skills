# 使用方法、本机命令与容器场景

这个文件只在以下情况按需读取：

- 用户明确问“这个 skill 怎么用”
- 用户问“命令怎么写”
- 用户问“在命令行里怎么跑”
- 用户问“在容器里怎么执行”
- 当前任务需要给出本机或容器内验证命令

默认不要把本文件全部内容搬进回答；只抽取与当前场景直接相关的几行。

## 最短使用说明

当用户只是想知道“怎么用”时，优先先给这 3 到 6 行，而不是一上来展开长篇迁移分析：

- 你可以直接给我 `Triton/CUDA` 代码、`PyTorch` 参考实现、文件路径，或报错/性能日志。
- 最好同时告诉我运行环境：本机命令行、已有容器、CI，或只需要我生成代码不执行。
- 如果你有偏好，也一起说：`最小 diff 迁移`、`文档风格`、`先跑通再优化`、`直接给优化版`。
- 我会按场景输出：迁移后的 `Triton-Ascend` 实现、最小验证脚本、执行命令、优化说明。

## 推荐提问方式

### 场景 1：给文件路径

```text
把 /path/to/kernel.py 里的 Triton CUDA kernel 迁到 Ascend，先给最小 diff 版，并附最小验证脚本。
```

### 场景 2：直接贴代码

```text
下面这段 PyTorch 算子改写成 Triton-Ascend，要求 1D grid，顺便检查 UB 和 dtype 退化。
```

### 场景 3：已有报错或性能问题

```text
这个 Triton-Ascend kernel 在 NPU 上报 UB overflow，帮我定位问题并直接给优化后的版本和验证脚本。
```

### 场景 4：容器内执行

```text
我在容器里跑 triton-ascend，帮我把 /workspace/op.py 里的 kernel 迁移掉，并给出容器内执行验证的命令。
```

### 场景 5：只要代码，不执行

```text
把这段 Triton kernel 迁到 Ascend，只要最终代码和验证脚本，不需要你实际执行。
```

## 环境识别规则

先判断用户希望你按哪种环境产出命令：

- 本机命令行
- 已有容器
- CI 或远端批处理环境
- 仅生成代码，不执行

按这个顺序推断：

1. 用户是否给了容器名、镜像名、`docker exec`、`docker run`、容器内路径
2. 用户是否给了本机目录、当前 shell 命令、当前工作目录
3. 如果仍无法判断，再追问“你希望我按本机命令行还是容器内命令来写验证步骤？”

## 输出命令时的默认规则

- 如果用户要的是命令，不要先写大段原理，先给最小可执行命令。
- 如果用户明确在容器里执行，优先给容器内路径和容器内命令，不要默认让用户在宿主机直接运行 Python。
- 如果当前环境不适合真实执行，也要继续给出命令模板，并明确“以下命令未在当前环境实际执行”。
- 如需安装或检查依赖，只给最小必要命令，不展开成长篇环境搭建教程，除非用户明确要求。

## 本机场景模板

下面这些是优先给出的最小命令模板，可按当前任务替换路径和脚本名。

### 进入目录再执行

```bash
cd /path/to/project
python verify_kernel.py
```

### 指定设备或环境变量

```bash
export ASCEND_RT_VISIBLE_DEVICES=0
python verify_kernel.py
```

如果当前工程已经有固定环境变量命名，优先沿用工程现有写法，不要生造新变量名。

## 容器场景模板

### 先进入容器再执行

```bash
docker exec -it <container> bash
cd /path/in/container
python verify_kernel.py
```

### 直接一次性执行

```bash
docker exec -it <container> bash -lc 'cd /path/in/container && python verify_kernel.py'
```

### 用户只给了 `docker run`

如果用户只给了启动命令或镜像名，但没有给容器内代码路径，先基于现有上下文推断；推断不出来时，再追问：

```text
你现在希望我按容器内哪个工作目录来写验证命令？
```

## 回答顺序建议

当用户问“怎么用”时，优先按这个顺序回答：

1. 用 3 到 6 行说明用户该提供什么
2. 点明当前按本机还是容器场景处理
3. 给 1 到 3 条最贴近当前问题的示例问法或命令
4. 再进入迁移、验证、优化正文

当用户问“命令怎么写”时，优先按这个顺序回答：

1. 最小可执行命令
2. 一句说明命令在哪个环境执行
3. 如有必要，再补充前置条件

## 不要这样做

- 不要每次都把本机、容器、CI 三套说明全写出来
- 不要在用户只问“怎么用”时就展开完整优化综述
- 不要把宿主机路径和容器内路径混写
- 不要声称命令已执行，除非你确实已经执行并拿到了结果
