# AI4S Profiling Analysis Checklist

本参考文档补充 AI for Science 模型在昇腾上的 Profiling 采集与分析要点。

## 采集级别选择

| 目标 | 级别 | 说明 |
|------|------|------|
| 快速看整体耗时分布 | L0 | 优先用于首次定位 |
| 看算子级热点和 shape | L1 | 适合训练或推理热点分析 |
| 看调用栈与内存问题 | L2 | 只在问题已经收敛后启用 |

## 采集前检查

- 先确保业务脚本可以在不采集的情况下稳定跑通。
- 训练场景优先跳过前若干 step，避开编译与数据预热噪声。
- 推理场景确认输入固定、batch 稳定，避免多轮结果不可比。
- 提前确认 trace 输出目录有足够磁盘空间。

## 推荐输出物

- `trace_view.json` 或 TensorBoard 目录。
- 一份 step 级别耗时摘要。
- 一份热点算子列表：算子名、shape、占比、调用次数。
- 一份结论：更偏向 hostbound、computing 还是 communication。

## AI4S 模型重点关注项

- 大模型或蛋白结构模型：重点看 host 到 device 下发节奏和图编译抖动。
- 扩散模型：重点看 attention、采样循环和随机数相关开销。
- 图网络模型：重点看稀疏算子和第三方扩展库带来的 host 开销。
- TensorFlow Community 路线：重点看 `npu_device` 初始化后是否真的落到 NPU。

## 结论写法建议

1. 先给出采集范围和采集级别。
2. 再给出最主要的瓶颈类型。
3. 最后给出下一步建议：继续下钻哪类 trace，还是回到代码层做适配。
