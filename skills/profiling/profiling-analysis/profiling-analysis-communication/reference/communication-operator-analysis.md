# 通信算子异常分析子技能

## 功能概述

该功能用于处理主分析Skill判定的非快慢卡问题的通信算子异常情况，直接建议用户查看plog日志以进一步分析通信问题。

## 核心功能

1. **日志检查建议**：提示用户检查plog日志以分析通信异常
2. **异常处理指导**：提供plog日志检查方法和异常处理建议
3. **优化建议**：提供通用的通信性能优化建议

## 处理流程

1. **接收通知**：接收主分析Skill的非快慢卡通信异常判定
2. **日志检查建议**：提示用户检查plog日志以进一步分析通信问题
3. **异常处理指导**：
   - 若plog中有hccl异常 → 联系接口人协助解决
   - 若plog中无异常 → 通信性能正常

## 使用方式

该功能由主分析Skill自动调用，无需用户手动执行。当主分析Skill判定为非快慢卡问题的通信算子异常时，会自动触发该功能并提示用户查看plog日志。

## PLOG日志检查建议

1. **检查hccl异常**：
   ```bash
   grep -i "error\|exception\|fail" *.plog | grep -i "hccl"
   ```

2. **检查通信相关错误**：
   ```bash
   grep -i "communication\|network\|timeout" *.plog
   ```

3. **错误类型判断**：
   - HcclTimeOutError：通信超时，可能是网络问题
   - HcclInternalError：内部错误，可能需要升级固件
   - HcclCommError：通信初始化失败，检查配置

## 优化建议

### 通信性能优化

1. **增加通信与计算重叠**：
   ```python
   # 示例：使用异步通信
   comm_handle = torch.distributed.all_reduce(tensor, async_op=True)
   # 执行计算任务
   tensor = computation(tensor)
   # 等待通信完成
   comm_handle.wait()
   ```

2. **优化数据传输**：
   - 减少不必要的数据传输
   - 优化数据格式和结构

3. **调整通信策略**：
   - 根据集群规模选择合适的通信算法
   - 调整批处理大小和并行度

4. **硬件与网络检查**：
   - 检查网络连接和带宽
   - 确认硬件配置一致性
